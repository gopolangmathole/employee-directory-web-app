Employee Directory Api Documentation

Employee -Read data of an employee

Get an points allow user or developers to get a single employee or a list of employees.

Advanced Doc
GET

click the following path to get a simple user based on the default employee id provided.

/api/v1/employees/{id}

The above rest-end will fetch a single employee based on the id given.

/api/v1/employees

The above rest-end will fetch all the available employees which are currently available.

Parameters

Field Type Description
id int The employee id.

Success 200

Field Type Description
id int The employee id.
firstName String The employee's first name.
lastName String The employee's last name.
employmentContract String Employee's contact with the company
gender String The employee's sex.
email String The employee's email address.
image File The employee's image.
address String The employee's current address

Errors

Status Code Description
404 Not found
400 Bad request
500 Internal server issue

Employee - Update an existing employee's image

The post request gives you full control of modifying and changing employee's image.
N.B When using postman make sure the header is empty and left as content-type without any type.
on the body, give a key value of imageFile and choose file as type, remember to specify the id of the user you want to change.

PUT

The following link should be used on the put mapping. The id should be of the email they want to update.

/api/v1/employees/{id}

Parameter

Field Type Description
id int The employee id.
imageFile File Upload employee's image through any client.
